# Day15｜MQ 核心概念（面试专用·纯记忆版）
**目标**：面试时能清晰说出 MQ 价值、场景与核心问题，**无需安装、无需启动、无需写代码**。

---

## 一、MQ 核心价值（3 句话背熟，必答）
1. **异步通信**：将同步等待的下游操作改为异步通知，显著提升接口响应速度，降低用户等待时长。
2. **服务解耦**：服务间通过消息传递事件，不再直接调用接口，一方升级或故障不影响另一方，提升系统弹性。
3. **流量削峰**：高并发请求先进入消息队列缓冲，消费端按能力匀速处理，避免下游服务被突发流量压垮。

---

## 二、微服务高频业务场景（1 个核心案例，面试直接说）
### 核心场景：订单创建 → 库存扣减
- **传统同步**：订单服务直接调用库存服务，库存服务挂了 → 订单创建失败，用户体验差。
- **MQ 异步**：订单创建成功后发送消息到 MQ，立即返回用户；库存服务监听消息，异步扣减库存。
- **核心优势**：解耦、高可用、提升下单成功率与响应速度。

## 三、面试必问 4 题（简短标准答案，直接背诵）
| 面试问题 | 标准答案（精简版） |
| :--- | :--- |
| **为什么要用 MQ？** | 异步提速、服务解耦、流量削峰。 |
| **使用 MQ 会带来哪些问题？** | 消息丢失、重复消费、消息乱序、消息堆积。 |
| **如何保证消息不丢失？** | 三步走：生产者确认（Confirm）、MQ 持久化（磁盘存储）、消费者手动 ACK（确认消费成功）。 |
| **如何解决重复消费？** | 业务实现幂等性：用唯一业务 ID（如订单号）做去重，结合 Redis 分布式锁或数据库唯一索引。 |
